> **Tauri 新手笔记**  
> - `tauri.conf.json` 是 Tauri 的「总控文件」：窗口大小、图标路径、dev/build 命令都在这里配  
> - `capabilities/default.json` 是 Tauri 2.0 安全模型：每个窗口需要明确声明权限才能调用原生 API  
> - `src-tauri/target/` 是 Rust 编译缓存，类似 node_modules，已在 .gitignore 中忽略  
> - 开发流程：终端开两个窗口 → `npm run dev`（启动 Vite 热重载）+ `npx tauri dev`（启动 Tauri 窗口）  
> - 图标更新：替换 `Lubina.svg` → 运行 `npx tauri icon Lubina.svg` → 自动更新 icons/ 下全部文件

### 前端 —— chat.js 对话管理

- 对话 CRUD：新建 / 切换 / 删除（localStorage 持久化）
- 首条消息自动截取前 20 字做标题
- SSE 流式接收，打字机效果实时渲染
- Markdown 渲染（marked.js）+ 数学公式预留（KaTeX）
- 停止生成（AbortController）
- 对话恢复：刷新页面后自动恢复历史对话

### 前端 —— api.js & bridge.js

- `api.js`：封装 fetch 调用，统一错误处理（ApiError 类）
  - `chatStream()`：SSE 流式聊天
  - `knowledgeAPI`：上传 / 提问 / 列表 / 删除
  - `configAPI`：配置读写
- `bridge.js`：Tauri 原生能力桥接，非 Tauri 环境静默降级
  - 文件对话框 / 剪贴板 / 系统通知 / Shell 打开
  - 文件系统读写 / 窗口控制

### 前端 —— 子页面

- **settings.html**：API Key（DeepSeek/OpenAI/Claude）+ 默认模型 + 主题切换 + 最大对话轮数。localStorage 持久化。
- **knowledge.html**：拖拽/点击上传区（PDF/PPTX/DOCX/TXT/MD），文件列表展示，RAG 工作流示意。上传逻辑占位，等后端就绪。
- **exercises.html**：统计卡片 + 科目筛选 + 错题卡片（题目/AI解析/错因/标签）。占位示例数据，P1 阶段正式开发。
